# Accessibility Audit Report — 2026-04-10

Editor page, all templates x all themes, edit mode.
Lighthouse score: **82/100** (6 failures). 99 WCAG AA contrast failures across the theme x layout matrix.

---

## 1. Critical: Headings invisible on `layout-full-dark` with light themes (ratio 1.11)

**All 6 light themes** (Studio Light, CUNY AI Lab, CUNY Light, Warm Academic, Slate Minimal, Forest) produce dark heading text (`#1a1a2e`) on the hardcoded dark background (`#0d1117`).

Root cause: `framework-preview.css:100` sets heading color to `var(--theme-text)` with `!important`, overriding the `color: #f0f0f0` inherited from the `layout-full-dark` rule at line 39.

**Fix:** Add a `layout-full-dark` heading override in `framework-preview.css`, similar to the title/divider/closing block:

```css
.slide-frame .slide[data-layout="layout-full-dark"] h1,
.slide-frame .slide[data-layout="layout-full-dark"] h2,
.slide-frame .slide[data-layout="layout-full-dark"] h3,
.slide-frame .slide[data-layout="layout-full-dark"] h4,
.slide-frame .slide[data-layout="layout-full-dark"] .heading {
  color: #f0f0f0 !important;
}
.slide-frame .slide[data-layout="layout-full-dark"] p,
.slide-frame .slide[data-layout="layout-full-dark"] .text-block,
.slide-frame .slide[data-layout="layout-full-dark"] .tiptap,
.slide-frame .slide[data-layout="layout-full-dark"] .tiptap p {
  color: rgba(240,240,240,0.65) !important;
}
```

Also apply same logic in `slide-html.ts:buildThemeCss()` for export.

## 2. Critical: Muted text fails AA on `layout-full-dark` (light themes, ratio 3.56)

Same root cause. `--theme-text-muted` for light themes is `~#6a6a7a` on `#0d1117`. Fix with same override above.

## 3. High: Accent color `#2FB8D6` fails AA on all light backgrounds

Affects Studio Light, CUNY AI Lab, CUNY Light across `layout-content`, `layout-split`, `layout-grid`.

| Background | Ratio |
|---|---|
| `#ffffff` (content) | 2.35 |
| `#e8eef6` (split) | 2.01 |
| `#e9f5f7` (grid) | 2.11 |

Used for: labels (`.label-cyan`), tip-box titles, comparison panel headings, links.

**Fix:** Darken accent for light themes. `#0e9ab5` passes AA (4.52:1 on white). Or switch to `color-mix()` with a darker variant when `--theme-bg` is light.

| Warm Academic | `#a78bfa` | 2.09 | all 7 layouts |
| Forest | `#059669` | 2.04 | title/divider/closing |

| Midnight | `#6366f1` | 2.56 | title/divider/closing, grid |
| Forest | `#34d399` (accent) | 1.65 | split, grid, content |

**Fix:** Secondary/accent colors need per-layout awareness. On primary-bg slides (title/divider/closing), use a lightened variant. On light content slides, darken them.

## 5. High: Editor chrome contrast failures (Lighthouse)

These are theme-independent — they're in the app UI surrounding the canvas:

| Element | FG | BG | Ratio | Required |
|---|---|---|---|---|
| "Wiz" brand text | `#5a8fd4` | `#ffffff` | 3.32 | 4.5 |
| Left tab buttons (Chat/Slides) | `#3b73e6` | `#ffffff` | 4.39 | 4.5 |
| Mode switcher (Edit/View) | `#3b73e6` | `#eff4fd` | 3.98 | 4.5 |
| "No slide selected" placeholder | `#6b7280` | `#f1f5f9` | 4.41 | 4.5 |
| Right tab buttons | `#3b73e6` | `#ffffff` | 4.39 | 4.5 |

**Fix:** Darken `#3b73e6` to `#2d5cc7` (passes 5.5:1 on white). Darken `#5a8fd4` to `#3a6fb4` for brand text.

## 6. High: 16 template card buttons have no accessible name

Every `.template-card` button in the Templates panel has no `aria-label`, `title`, or visible text that screen readers can announce. They contain only thumbnail images.

**Fix:** Add `aria-label="{template name}"` to each template card button.

---

## 7. Medium: Missing `<main>` landmark

The editor page has no `<main>` element or `role="main"`. Screen readers cannot navigate to the primary content area.

**Fix:** Wrap the center panel (canvas area) in a `<main>` tag.

---

## 8. Medium: Model selector `<select>` has no label

The AI model dropdown (`.model-selector select`) has no associated `<label>`, `aria-label`, or `title`.

**Fix:** Add `aria-label="AI model"` to the select element.

---

## 9. Medium: TipTap chat editor has no accessible name

The chat input TipTap editor (`div.tiptap[role="textbox"]`) has no `aria-label`. Screen readers announce it as a generic text input.

**Fix:** Add `aria-label="Chat message"` to the TipTap mount.

---

## 10. Medium: Form fields missing id/name (Chrome issue)

2 console issues: form field elements without `id` or `name` attributes.

---

## 11. Medium: No skip link

No skip-to-content link exists. Keyboard users must tab through the entire left panel to reach the canvas.

**Fix:** Add a skip link targeting the canvas area.

---

## 12. Low: Undersized tap targets (< 44px)

48 buttons are below the 44x44px recommended minimum. Worst offenders:

| Control | Size | Location |
|---|---|---|
| Module actions (kebab) | 16x16 | Canvas edit controls |
| Mention in chat (@) | 20x20 | Template panel |
| Format toolbar buttons (B/I/S/list) | 22x22 | Chat input |
| Reset chat | 47x21 | Chat header |
| + Module | 76x24 | Canvas |
| Nav buttons (prev/next) | 27x22 | Canvas toolbar |
| Mode buttons (Edit/View) | ~40x22 | Canvas toolbar |
| Tab buttons (Files/Templates/...) | ~50x30 | Right panel |

**Fix:** Increase minimum touch target to 44x44 via padding. The visual size can remain smaller if the hit area is padded.

---

## 13. Low: Broken ARIA tab pattern

6 `[role="tab"]` elements exist across 2 `[role="tablist"]` containers, but 0 `[role="tabpanel"]` elements exist. The tab/tabpanel association is incomplete.

**Fix:** Add `role="tabpanel"` to the content area each tab controls, with `aria-labelledby` pointing to the tab.

---

## 14. Low: Heading hierarchy skips h2

The page has `h1` (slide content) then jumps to `h3` (template section headings, subtitle). No `h2` exists.
